# P23 threat model — personal macOS local application

**Scope:** local server, browser UI, project files, imported media, executable compositions, bridge/CLI, render workers, caches, logs, outputs, dependencies, and release packaging.  
**Default posture:** loopback-only, token-authenticated, least privilege, deny by default, immutable evidence, imported execution fail-closed.

| Asset / boundary                   | Threat                                                                                                               | Control authority                                                                                                                                                             | Mandatory evidence                                             | Residual risk                                                                                                                    |
| ---------------------------------- | -------------------------------------------------------------------------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | -------------------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------- |
| Local HTTP server                  | LAN exposure, DNS rebinding, forged Host/Origin, CSRF mutation, token guessing                                       | Loopback-only bind; loopback Host validation; exact origin allowlist; 256-bit session token; matching anti-CSRF header for browser mutations; restrictive CORS                | Request-security unit/integration and non-loopback bind tests  | A compromised same-user process may read local runtime state; OS user-account integrity remains assumed.                         |
| Browser UI                         | Navigation, popup, download, external protocol, broad file URL, local-service probing, clipboard or permission abuse | CSP, COOP/CORP, Permissions-Policy, same-origin navigation/service policy, approved-root file policy, explicit-gesture clipboard, deny popups/downloads/protocols/permissions | Browser-policy tests plus response-header integration          | Browser/OS vulnerabilities are outside application control; managed browser and dependency updates require re-review.            |
| Project and external files         | Traversal, prefix trick, symlink escape, unauthorized write, unsafe temp/output root                                 | Lexical traversal rejection followed by canonical realpath containment and access-mode allowlists                                                                             | Existing/prospective path and symlink adversarial tests        | TOCTOU is reduced but not eliminated without descriptor-relative OS APIs; workers use read-only roots where possible.            |
| Executable compositions            | Imported code promoted by output success; undeclared capabilities                                                    | Exact source-hash trust record; explicit promotion review; imported support requires project opt-in plus current isolation evidence                                           | Trust/promotion/preflight/receipt tests                        | Human review can miss malicious logic; imported content remains isolated even after successful renders until explicit promotion. |

## Imported execution decision

Imported executable content is not generally enabled by a successful test or render. It is permitted only for the exact macOS policy identity whose adversarial evidence is current, while every artifact retains matching trust, environment, worker-pool, browser-profile, temporary-root, and cache provenance. Missing or deprecated enforcement disables imported execution.
