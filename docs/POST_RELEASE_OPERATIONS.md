# Post-release operations

Triage by release identity, schema/revision, environment fingerprint, correlation ID, exact frame/range, and fixture. Request only redacted local support-bundle previews; never collect media, executable source, secrets, cookies, or persistent profiles.

Security reports trigger containment, version analysis, dependency/license review, a scoped fix, full gates, and user guidance. Validate backups regularly and before migration/upgrades. Rollback restores an immutable compatible release and never silently downgrades schema.

Review dependency alerts, engine releases, browser/FFmpeg/macOS changes, licenses, capability drift, budgets, and support incidents on a release cadence. Public distribution, commercialization, team growth, wrapper/cloud work, or automation-scale changes trigger new security/license/support ADR review.
