# Test evidence convention

Every accepted task records:

1. Task ID and exact definition of done.
2. Changed implementation or frozen contract.
3. Commands and machine-readable results.
4. Strict environment or compatible-preview identity as applicable.
5. Manual/visual evidence when output meaning cannot be proven structurally.
6. Requirement, ADR, risk, and release-gate traceability.
7. Explicit exception, residual risk, downstream task, and owner when anything remains.

Use `docs/templates/task-evidence.md`. Evidence must distinguish rendered, QA-passed, approved, and delivered states. A successful encode is only a render result.
